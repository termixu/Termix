> **Версия:** 1.0
>
> **Дата:** Апрель 2026
>
> **Назначение:** Методика проведения квизов и тестов на занятиях
>
> **Принцип:** Не контроль и оценки, а игра и повторение. Не стресс, а азарт.

---

## 🧠 ФИЛОСОФИЯ ТЕСТИРОВАНИЯ

Обычный подход: тест → оценка → «я не знаю» → страх → ребёнок боится ошибаться.

Подход Termix Academy: тест → игра → «я знаю!» → жетоны и баджи → ребёнок хочет ещё.

Квизы в Termix Academy — это не проверка знаний. Это способ:
- Повторить материал в игровой форме.
- Дать ребёнку ощущение «я это знаю».
- Собрать обратную связь для наставника: какие темы усвоены, а где пробелы.
- Разогреть группу перед занятием или завершить занятие на позитиве.

Никаких оценок. Только жетоны, баджи и строчки в журнале наставника.

## 🛠️ ИНСТРУМЕНТЫ ДЛЯ ПРОВЕДЕНИЯ КВИЗОВ

### Plickers — для офлайн-занятий без телефонов

**Где взять:** plickers.com + приложение на телефон.

**Как работает:**
- Ты распечатываешь карточки с QR-кодами (по одной на ребёнка).
- Показываешь вопрос на проекторе или экране.
- Дети поднимают карточки нужной стороной (A, B, C, D).
- Ты сканируешь класс камерой телефона — ответы собираются мгновенно.

**Плюсы:**
- Детям не нужны телефоны и компьютеры.
- Работает как магия: комната, карточки, камера.
- Бесплатно для групп до 63 человек.

**Минусы:**
- Нужно распечатать карточки (один раз).
- Вопросы готовятся заранее на сайте Plickers.

**Идеально для:** младших групп (6–10 лет), офлайн-занятий, IT-лагеря.

---

### Open eClass — для самостоятельного сервера

**Где взять:** open-eclass.org — бесплатно, открытый исходный код.

**Как работает:**
- Устанавливается на твой ноутбук или сервер.
- Дети подключаются с любых устройств по локальной сети (Wi-Fi).
- Есть квизы, опросы, тесты с разными типами вопросов.

**Плюсы:**
- Всё под твоим контролем.
- Не зависит от интернета.
- Можно использовать в локальной сети (например, в IT-лагере).

**Минусы:**
- Нужно установить и настроить (технически несложно, но требует времени).
- Требует локальной сети.

**Идеально для:** IT-лагеря, занятий в классе, где у детей есть телефоны или планшеты.

---

### Moodle — если уже есть или планируется

**Где взять:** moodle.org — бесплатно. Установка на сервер (можно локальный через XAMPP).

**Как работает:**
- Полноценная система управления обучением (LMS).
- Тесты с разными типами вопросов, ограничением по времени, случайным порядком вопросов и ответов.
- Автоматическая проверка и журнал оценок.

**Плюсы:**
- Мощный инструмент с кучей настроек.
- Можно использовать для всего курса, не только для квизов.

**Минусы:**
- Тяжеловесный. Ставить ради одних квизов — избыточно.
- Требует настройки и администрирования.

**Идеально для:** полноценной платформы Termix Academy (будущее).

---

### Quizizz и Kahoot — для быстрых онлайн-квизов

**Где взять:** quizizz.com, kahoot.com — бесплатно.

**Как работает:**
- Создаёшь квиз на сайте.
- Дети подключаются по коду с телефонов или компьютеров.
- Вопросы на экране, варианты ответов на устройствах детей.
- Есть музыка, таймер, рейтинг после каждого вопроса.

**Плюсы:**
- Очень нравится детям (геймификация, музыка, соревнование).
- Быстро создать и запустить.
- Можно дать как домашнее задание.

**Минусы:**
- Требует интернета у всех участников.
- Бесплатная версия имеет ограничения.

**Идеально для:** онлайн-занятий, домашних заданий, разогрева в начале занятия.

## 🎮 КОГДА И КАК ИСПОЛЬЗОВАТЬ КВИЗЫ

### Разогрев в начале занятия (5–7 минут)

**Цель:** Разбудить, включить мозг, повторить прошлую тему.

**Формат:** 3–5 вопросов по прошлому занятию. Короткие, весёлые, с картинками Termix'а.

**Пример:**
- «Какую команду мы изучили на прошлом занятии?»
- «Что делает git init?»

**Награда:** 1 жетон за каждый правильный ответ. Первый ответивший — дополнительный жетон.

---

### Повторение в середине модуля (10–15 минут)

**Цель:** Проверить усвоение материала, найти пробелы.

**Формат:** 8–10 вопросов по темам модуля. Разные типы: выбор ответа, открытый вопрос, «найди ошибку в коде».

**Пример:**
- «Что выведет этот код? print(2 + 2 * 2)»
- «Где ошибка? git comit -m "message"»

**Награда:** жетоны за правильные ответы. За 8+ правильных — бадж «Знаток модуля».

---

### Финальный квиз в конце курса (20–30 минут)

**Цель:** Подвести итог, дать ощущение завершённости, выявить лучших.

**Формат:** 15–20 вопросов по всему курсу. Можно в формате командной игры.

**Пример:**
- Турнир «Пиратский Квиз»: команды по 2–3 человека.
- Вопросы на скорость и точность.
- Победившая команда — бадж «Квиз-капитан».

**Награда:** баджи, дополнительные жетоны, грамота.

## 📋 ТИПЫ ВОПРОСОВ

### Множественный выбор

Самый частый формат. 4 варианта ответа, один правильный.

Пример:
```

Что делает команда pwd в терминале?
A) Удаляет файл
B) Показывает текущую папку (правильный)
C) Копирует файл
D) Перезагружает компьютер

```

### Открытый вопрос

Нужно ввести ответ текстом.

Пример:
```

Напиши команду для создания папки "pirate-fort".
Ответ: mkdir pirate-fort

```

### Найди ошибку

Показываем фрагмент кода с ошибкой. Нужно найти и исправить.

Пример:
```

Найди ошибку:
print("Привет, Termix!)

```
(Пропущена закрывающая кавычка.)

### Сопоставление

Даны два столбца. Нужно соединить пары.

Пример:
```

Команда       | Что делает
git init      | Создать репозиторий
git add .     | Добавить все файлы
git commit    | Сохранить изменения
git push      | Отправить на сервер

```

### Горячие клавиши

Самый быстрый формат. Вопрос и две кнопки: ✅ и ❌.

Пример:
```

"git push отправляет код на сервер"
✅ (Да) или ❌ (Нет)

```

## 🎨 ОФОРМЛЕНИЕ КВИЗОВ

### Визуальный стиль

- Тёмный фон (#0a0a0a).
- Зелёный текст вопросов (#00ff41).
- Картинки Termix'а для позитивных эмоций.
- Мем-реакции на правильные и неправильные ответы.

### Meme-реакции от Termix'а

- **Правильный ответ:** «Бип-бип! В яблочко!», «Мур-мур-молодец!», «Хвост зеленеет!».
- **Неправильный ответ:** «K-k-k... Почти!», «Хвост желтеет...», «Попробуй ещё раз, юнга!».
- **Завершение квиза:** «Бип! Квиз пройден! Жетоны начислены!».

## 📊 АНАЛИТИКА ДЛЯ НАСТАВНИКА

После каждого квиза наставник видит:
- Какие вопросы вызвали наибольшие затруднения.
- Кто из учеников отвечал лучше всех.
- Какие темы нужно повторить.

Эта информация заносится в журнал (см. `core-failures.md` для связи с системой «Сбои Ядра»). Если ученик регулярно ошибается на одной теме — это флаг для наставника: «Нужно повторить».
