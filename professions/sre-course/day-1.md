📚 Теория минимумом, практика — максимум

В Linux всё — файлы. Логи, настройки, программы, даже диски — всё это файлы. Если вы научитесь с ними работать, вы освоили 50% работы инженера сопровождения.

---

1. pwd — где я?

Когда вы открываете терминал, вы находитесь в какой-то папке. Команда pwd (print working directory) показывает полный путь.

Пример:

```bash
user@host:~$ pwd
/home/user
```

Тильда (~) означает вашу домашнюю папку. Обычно это /home/имя_пользователя.

---

2. ls — что внутри?

ls (list) показывает содержимое папки.

Полезные ключи:

· ls -l — подробный список (права, владелец, размер, дата)
· ls -a — показать скрытые файлы (начинаются с точки)
· ls -la — комбинация двух (чаще всего используется)
· ls -lh — человекочитаемые размеры (1K, 2M, 3G)
· ls -lt — отсортировать по дате изменения (сначала новые)
· ls -ltr — отсортировать по дате (сначала старые)

Пример:

```bash
user@host:~$ ls -la
total 48
drwxr-x--- 5 user user 4096 May 1 10:00 .
drwxr-xr-x 3 root root 4096 May 1 09:00 ..
-rw------- 1 user user  120 May 1 08:00 .bash_history
-rw-r--r-- 1 user user  220 May 1 08:00 .bash_logout
```

Что видим?

· d — директория (папка)
· - — обычный файл
· rwx — права (чтение, запись, выполнение)
· user user — владелец и группа
· 4096 — размер в байтах
· May 1 10:00 — дата изменения

---

3. cd — перейти в другую папку

cd (change directory) перемещает вас по файловой системе.

Примеры:

```bash
cd /var/log          # перейти в /var/log (абсолютный путь)
cd logs              # перейти в папку logs (относительно текущей)
cd ..                # подняться на уровень выше
cd ../..             # подняться на два уровня
cd ~                 # вернуться в домашнюю папку
cd -                 # вернуться в предыдущую папку
```

---

4. file — что за зверь?

Не всё то текст, чем кажется. file определяет тип файла.

Пример:

```bash
user@host:~$ file /etc/passwd
/etc/passwd: ASCII text

user@host:~$ file /bin/ls
/bin/ls: ELF 64-bit LSB executable
```

---

5. Просмотр содержимого файлов

cat — вывести весь файл в терминал (хорошо для маленьких файлов)

```bash
cat /etc/hostname
```

less — постраничный просмотр (навигация стрелками, q — выход, / — поиск)

```bash
less /var/log/syslog
```

head — первые 10 строк (можно указать сколько: head -n 20)

```bash
head /var/log/syslog
```

tail — последние 10 строк (можно указать сколько)

```bash
tail /var/log/syslog
```

tail -f — следить за файлом в реальном времени (очень часто используется!)

```bash
tail -f /var/log/syslog
```

Нажмите Ctrl+C для выхода.

---

6. wc — посчитать строки, слова, байты

wc (word count) считает строки (-l), слова (-w), байты (-c).

Пример:

```bash
user@host:~$ wc -l /var/log/syslog
542 /var/log/syslog
```

Сколько в логе строк ошибок? grep -c (об этом завтра).

🔧 Практическое задание 2. Анализируем файл паролей

Внимание: файл /etc/passwd содержит информацию о пользователях, но не пароли (они в /etc/shadow).

Что делаем:

1. Перейдите в корневую папку: cd /
2. Посмотрите содержимое: ls -la
3. Посмотрите количество строк в файле пользователей: wc -l /etc/passwd
4. Посмотрите первые 5 строк: head -n 5 /etc/passwd
5. Что видите? Формат: username:x:UID:GID:описание:домашняя_папка:shell
6. Найдите пользователя root: grep "root" /etc/passwd (ещё не учили, но попробуйте)

🔧 Практическое задание 3. Наблюдаем за логами в реальном времени

Что делаем:

1. Откройте два терминала (или вкладки в терминале)
2. В первом терминале выполните: tail -f /var/log/syslog
3. Во втором терминале выполните что-нибудь, что создаст запись в логе, например:
   · sudo systemctl restart cron (если спросит пароль — введите)
   · Или просто ls /root (у вас не будет доступа, но лог зафиксирует ошибку)
4. Вернитесь в первый терминал. Появились новые строки?
5. Нажмите Ctrl+C в первом терминале, чтобы остановить tail

🔧 Практическое задание 4. Создаём свой файл и работаем с ним

Что делаем:

1. Перейдите в домашнюю папку: cd ~
2. Создайте новую папку: mkdir practice
3. Перейдите в неё: cd practice
4. Создайте текстовый файл с помощью nano (простого редактора):

1. Напишите 5-6 строк текста. Например:

1. Сохраните: Ctrl+O, Enter. Выйдите: Ctrl+X

3. Посмотрите первую строку: head -n 1 myfile.txt
4. Посмотрите последние две строки: tail -n 2 myfile.txt
5. Посчитайте строки: wc -l myfile.txt

🧠 Шпаргалка (можно сохранить)

```bash
# Где я?
pwd

# Что внутри?
ls           # просто имена
ls -l        # подробно
ls -la       # подробно + скрытые
ls -lh       # подробно + размеры в K/M/G
ls -ltr      # подробно + сортировка по дате (старые внизу)

# Перемещение
cd /var/log  # абсолютный путь
cd logs      # относительный
cd ..        # наверх
cd ~         # в домашнюю
cd -         # обратно в предыдущую

# Просмотр файлов
cat file.txt           # весь файл
less file.txt          # постранично (q выход)
head file.txt          # первые 10 строк
head -n 20 file.txt    # первые 20
tail file.txt          # последние 10
tail -f file.txt       # следить за изменениями

# Счёт
wc -l file.txt    # количество строк
wc -w file.txt    # количество слов
wc -c file.txt    # количество байт

# Тип файла
file /etc/passwd
```

🚨 Типичные ошибки новичков

1. Забываю поставить пробел между командой и аргументом
   · ❌ ls-la
   · ✅ ls -la
2. Регистр важен
   · ❌ LS -LA
   · ✅ ls -la
3. Не могу найти файл, потому что не в той папке
   · Перед поиском всегда pwd и ls
4. Не могу выйти из less
   · Нажмите q
5. Не могу остановить tail -f
   · Нажмите Ctrl+C
