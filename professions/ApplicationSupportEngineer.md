## 🎯 ЧТО ДЕЛАЕТ APPLICATION SUPPORT ENGINEER

Application Support Engineer — это мост между пользователями, которые жалуются «у меня не работает», и разработчиками, которые говорят «у меня всё работает». Он разбирается в логах, базах данных, настройках серверов и коде. Он находит корень проблемы и либо чинит сам, либо передаёт разработчикам с точным описанием бага.

**Типичные задачи:**
- Принимать заявки от пользователей (тикеты).
- Анализировать логи сервера и приложения.
- Диагностировать проблемы в базах данных.
- Писать простые скрипты для автоматизации.
- Настраивать окружения (development, staging, production).
- Мониторить работоспособность систем.
- Документировать решения и инциденты.

**Где работает:**
- В IT-компаниях (поддержка своих продуктов).
- В банках и телекоме (поддержка критичных систем).
- На фрилансе (поддержка небольших сервисов).
- Удалённо (многие инциденты решаются из дома).

## 🗺️ ПУТЬ В TERMIX ACADEMY

### Этап 1: Фундамент (13–15 лет)

Курсы Termix Academy, с которых начинается путь:

- **Terminal (Bash)** — командная строка. Просмотр логов через `tail -f`, поиск через `grep`, навигация по серверу.

- **Linux** — операционная система. Большинство серверов работают на Linux. Пользователи, права, процессы, systemd, nginx.

- **SQL и PostgreSQL** — базы данных. SELECT с JOIN, поиск аномалий, проверка целостности данных.

- **Python** — скрипты автоматизации. Обработка логов, генерация отчётов, мелкие утилиты.

- **Git и GitHub** — контроль версий. Понимание, какая версия приложения где развёрнута.

**Что умеет после этапа:**
- Подключиться к серверу по SSH и посмотреть логи.
- Написать SELECT-запрос для проверки данных.
- Написать скрипт для парсинга логов.
- Понимать, как работает веб-сервер.

**Первые проекты для портфолио:**
- Скрипт-монитор: проверяет доступность сайта и пишет в лог.
- SQL-запросы для поиска аномалий в тестовой базе.
- Bash-скрипт для сбора логов с сервера.

### Этап 2: Инструменты поддержки (14–16 лет)

- **Docker** — контейнеризация. Поднять тестовое окружение одной командой.

- **GitHub Actions** — автоматизация. Настроить алерты и автопроверки.

- **VS Code** — редактор. Быстрый просмотр конфигов и скриптов.

- **AI-промптинг** — использование ChatGPT для диагностики ошибок.

**Что умеет после этапа:**
- Поднять тестовое окружение в Docker.
- Настроить мониторинг и алерты.
- Использовать AI для быстрой диагностики.

**Первые проекты для портфолио:**
- Docker-compose для тестового окружения Termix Academy.
- CI/CD-пайплайн с проверкой работоспособности после деплоя.
- Мини-система тикетов (терминальная).
