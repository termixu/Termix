# 🤖 КУРС «AI-ПРОМПТИНГ: ЗАКЛИНАТЕЛИ НЕЙРОСЕТЕЙ»

> **Версия:** 1.0
>
> **Возраст:** 12–16 лет
>
> **Длительность:** 8 занятий по 75–90 минут
>
> **Проводник:** Termix — кибер-пиратский кот, Хранитель Ядра
>
> **Принцип:** Не бояться ИИ, а управлять им. Не просто копировать ответы, а создавать шедевры через правильные запросы.

---

## 🧠 ФИЛОСОФИЯ КУРСА

Обычный подход: нейросеть → «напиши реферат» → плохой ответ → «ИИ глупый».

Подход Termix Academy: нейросеть → «ты — пиратский историк, напиши отчёт о сокровищах в стиле XVIII века» → потрясающий ответ → «я умею управлять ИИ!».

AI-промптинг — это навык будущего. Умение правильно формулировать запросы к нейросетям (ChatGPT, DALL-E, Midjourney) становится важнее, чем умение печатать. Termix учит детей не просто «спрашивать у робота», а давать ему роли, примеры, ограничения и форматы. К концу курса ребёнок сможет генерировать код, картинки, сценарии и учебные материалы с помощью ИИ.

---

## 🎯 ЧЕМУ НАУЧИТСЯ РЕБЁНОК

- Понимать, как работают большие языковые модели (LLM).
- Формулировать эффективные промпты.
- Давать ИИ роль, контекст и ограничения.
- Использовать цепочки промптов (chain prompting).
- Генерировать код с помощью ИИ.
- Генерировать изображения (DALL-E, Midjourney, Stable Diffusion).
- Использовать ИИ для учёбы (объяснения, шпаргалки, конспекты).
- Понимать этику ИИ: галлюцинации, авторское право, конфиденциальность.

## 🗺️ КАРТА КУРСА

- **Занятие 1 — «Первое заклинание».** Что такое LLM. Как нейросеть «думает». Первый промпт.

- **Занятие 5 — «Код: ИИ-программист».** Генерация кода. Отладка через промпты.
- **Занятие 6 — «Изображения: ИИ-художник».** Генерация картинок. DALL-E, Midjourney.
- **Занятие 7 — «Учёба: ИИ-учитель».** Объяснение сложных тем. Конспекты. Шпаргалки.
- **Занятие 8 — «Галерея заклинаний».** Парад лучших промптов. Создание своей коллекции.

## 📖 ЗАНЯТИЕ 1: ПЕРВОЕ ЗАКЛИНАНИЕ

### 🎯 Цель

Понять, что такое LLM, как нейросеть генерирует текст, написать первый промпт.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «Бип! Добро пожаловать в школу заклинателей! Нейросеть — это джинн. Промпт — твоё желание. Погнали!»
- 5–15 мин: Что такое LLM. Обучение на текстах. Токены. Температура.
- 15–25 мин: Почему промпт важен. Garbage in — garbage out.
- 25–45 мин: Первый промпт. «Расскажи о себе». Сравниваем ответы с разными формулировками.
- 45–55 мин: Плохой vs Хороший промпт. «Напиши рассказ» vs «Напиши рассказ на 200 слов о пирате, который нашёл сундук с котом внутри».
- 55–60 мин: Итоги. Бадж «Первое Заклинание».

### 🛠️ Практика

Ребёнок открывает ChatGPT (или DeepSeek, или Claude). Пишет промпт:

**Плохой:**
```

Напиши рассказ.

```

**Хороший:**
```

Напиши короткий рассказ на 150 слов.
Главный герой — кот-пират Termix с зелёным кибер-глазом.
Он нашёл сундук, внутри которого оказался не золото, а портал в другое измерение.
Закончи рассказ неожиданным поворотом.

```

Сравнивает результаты. Обсуждает с наставником, почему второй ответ лучше.

### 🎮 Игровая механика

Игра «Испорченный телефон». Наставник даёт плохой промпт. Дети улучшают его. У кого лучший ответ от ИИ — жетон.

### 🏠 Миссия

- 🥉 Юнга: Написать 3 разных промпта на одну тему и сравнить ответы.
- 🥈 Боцман: Улучшить плохой промпт наставника и получить качественный ответ.
- 🥇 Штурман: Написать промпт, который генерирует текст в стиле Termix'а.

## 📖 ЗАНЯТИЕ 2: РОЛЬ И МАСКА

### 🎯 Цель

Освоить role prompting — давать ИИ роль для получения качественных ответов.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «ИИ может быть кем угодно! Пиратским капитаном, учителем, учёным. Дай ему роль — и ответ изменится!»
- 5–15 мин: Что такое role prompting. «Ты — ...» как главная фраза.
- 15–30 мин: Сравнение. Один вопрос с ролью и без.
- 30–50 мин: Практика. Даём роли: пиратский историк, повар на корабле, боцман.
- 50–70 мин: Системные промпты. Role + Context + Task.
- 70–80 мин: Практика: пишем системный промпт для Termix'а.
- 80–90 мин: Итоги. Бадж «Маска».

### 🛠️ Практика

Ребёнок пишет промпты с разными ролями:

**Без роли:**
```

Расскажи про пиратские корабли.

```

**С ролью:**
```

Ты — пиратский историк XVIII века.
Ты служил на трёх кораблях и знаешь всё о парусах и пушках.
Расскажи юнге о том, как устроен пиратский корабль.
Используй морские термины и говори как настоящий морской волк.

```

**Системный промпт для Termix'а:**
```

Ты — Termix, кибер-пиратский кот.
Твои особенности:

· Говоришь с «бип» и «мяу» в начале фраз.
· У тебя зелёный кибер-глаз и крюк из клавиши Enter.
· Ты учишь детей программированию.
· Отвечаешь с юмором, но по делу.
  Ответь на вопрос: «Что такое переменная в Python?»

```

### 🎮 Игровая механика

Игра «Угадай роль». Наставник зачитывает ответ ИИ. Дети угадывают, какая роль была задана.

### 🏠 Миссия

- 🥉 Юнга: Задать один вопрос с тремя разными ролями и сравнить ответы.
- 🥈 Боцман: Создать системный промпт для учителя математики.
- 🥇 Штурман: Создать персонажа-помощника для своей любимой игры.

## 📖 ЗАНЯТИЕ 3: ФОРМАТ И ПРИМЕРЫ

### 🎯 Цель

Освоить управление форматом ответа и few-shot prompting.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «Формат! Хочешь таблицу? Скажи. Хочешь список? Скажи. Хочешь JSON? Скажи! ИИ подчиняется формату!»
- 5–15 мин: Управление форматом. Markdown, таблицы, списки, JSON.
- 15–30 мин: Few-shot prompting. Дай пример — ИИ повторит структуру.
- 30–50 мин: Практика. Генерируем таблицы, списки, структуры.
- 50–70 мин: Zero-shot vs One-shot vs Few-shot.
- 70–80 мин: Создание шаблонов промптов.
- 80–90 мин: Итоги. Бадж «Форматировщик».

### 🛠️ Практика

**Управление форматом:**
```

Составь список из 5 пиратских команд Git.
Для каждой команды укажи:

· Название команды
· Что она делает
· Пиратскую аналогию (как это выглядит на корабле)
· Смешную фразу Termix'а про эту команду

Формат: таблица Markdown.

```

**Few-shot prompting:**
```

Вот как Termix объясняет команды:

Пример:
Команда: git init
Объяснение: Создать новый сундук для сокровищ.
Фраза: "Бип! Я создал этот пустой сундук. Клади сюда код!"

Теперь объясни в таком же стиле команды: git add, git commit, git push.

```

### 🎮 Игровая механика

Соревнование «Шаблон». Наставник: «Создай шаблон промпта для генерации карточек с пиратами. У кого структура лучше — жетон».

### 🏠 Миссия

- 🥉 Юнга: Сгенерировать таблицу с помощью промпта.
- 🥈 Боцман: Использовать few-shot prompting для стилизации ответа.
- 🥇 Штурман: Создать свой шаблон для генерации учебных карточек.

## 📖 ЗАНЯТИЕ 4: ЦЕПОЧКИ ЗАКЛИНАНИЙ

### 🎯 Цель

Освоить chain prompting — разбивку большой задачи на шаги.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «Одно заклинание — хорошо. Цепочка — могущество! Большие задачи решаем по шагам!»
- 5–15 мин: Что такое chain prompting. Зачем разбивать на шаги.
- 15–30 мин: Пример: создание игры. Шаг 1 — идея. Шаг 2 — механика. Шаг 3 — код.
- 30–50 мин: Практика. Пишем цепочку для создания веб-страницы.
- 50–70 мин: Отладка через цепочку. Ошибка — новый промпт с контекстом ошибки.
- 70–80 мин: Саморефлексия ИИ. «Проверь свой ответ и найди ошибки».
- 80–90 мин: Итоги. Бадж «Цепной Заклинатель».

### 🛠️ Практика

Ребёнок пишет цепочку промптов:

**Шаг 1 — Идея:**
```

Придумай простую браузерную игру для ребёнка 10 лет.
Тема: пираты. Игра должна работать в браузере на HTML+JS.
Опиши идею в 3 предложениях.

```

**Шаг 2 — Механика:**
```

Опиши механику этой игры подробно.
Что делает игрок? Как начисляются очки? Что является препятствием?

```

**Шаг 3 — Код:**
```

Напиши полный HTML+JS код этой игры.
Использу Canvas. Добавь комментарии на русском.

```

### 🎮 Игровая механика

Квест «Собери игру». Наставник: «Используй цепочку из 4 промптов, чтобы создать браузерную игру. У кого работает — жетон».

### 🏠 Миссия

- 🥉 Юнга: Создать цепочку из 2 промптов на любую тему.
- 🥈 Боцман: Создать цепочку из 4 промптов для создания веб-страницы.
- 🥇 Штурман: Отладить код с помощью цепочки промптов (специально сломанный код от наставника).

## 📖 ЗАНЯТИЕ 5: КОД — ИИ-ПРОГРАММИСТ

### 🎯 Цель

Освоить генерацию и отладку кода через промпты.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «ИИ умеет кодить! Но ему нужен чёткий приказ. Спецификация — наше всё!»
- 5–15 мин: Как просить код. Спецификация: язык, входные данные, выходные данные, пример.
- 15–30 мин: Генерация Python-функции по спецификации.
- 30–50 мин: Генерация HTML/CSS/JS страницы.
- 50–70 мин: Отладка. «Найди ошибку в этом коде». «Объясни, что делает эта строка».
- 70–80 мин: Улучшение кода. «Оптимизируй этот код». «Добавь комментарии».
- 80–90 мин: Итоги. Бадж «ИИ-Программист».

### 🛠️ Практика

**Спецификация:**
```

Напиши функцию на Python.

Название: count_treasure
Вход: список строк (названия предметов)
Выход: словарь, где ключ — предмет, значение — сколько раз он встретился
Пример:
Вход: ["золото", "рыба", "золото", "карта", "рыба", "рыба"]
Выход: {"золото": 2, "рыба": 3, "карта": 1}

Добавь комментарии к каждой строке.

```

**Отладка:**
```

Вот код. Он должен считать сумму чисел в списке, но выдаёт ошибку.
Найди ошибку и исправь:

def sum_list(nums):
total = 0
for i in range(len(nums) + 1):
total += nums[i]
return total

```

### 🎮 Игровая механика

Соревнование «Спецификация». Наставник: «Опиши функцию так, чтобы ИИ написал её правильно с первого раза». Чья спецификация точнее — жетон.

### 🏠 Миссия

- 🥉 Юнга: Сгенерировать простую функцию по спецификации.
- 🥈 Боцман: Сгенерировать веб-страницу с формой и стилями.
- 🥇 Штурман: Отладить 3 ошибки в коде через промпты.
