## 🧠 ФИЛОСОФИЯ КУРСА

Обычный подход: SQL → таблицы → SELECT FROM WHERE → скучно → «зачем это нужно?».

Подход Termix Academy: SQL → пиратская база данных → найди всех пиратов с золотом больше 100 → «я могу управлять данными!».

SQL — это язык, на котором говорят все базы данных мира. Каждый сайт, каждое приложение, каждая игра хранит информацию в таблицах. Termix ведёт юных капитанов данных от первого SELECT до сложных запросов с JOIN и GROUP BY. Всё на примере пиратской базы: корабли, сокровища, экипаж, рейды.

---

## 🎯 ЧЕМУ НАУЧИТСЯ РЕБЁНОК

- Понимать, что такое база данных и зачем она нужна.
- Писать запросы SELECT с фильтрацией WHERE.
- Сортировать данные с ORDER BY.
- Использовать функции: COUNT, SUM, AVG, MAX, MIN.
- Группировать данные с GROUP BY.
- Соединять таблицы через JOIN.
- Добавлять, изменять, удалять данные (INSERT, UPDATE, DELETE).
- Проектировать простую базу данных.

## 🗺️ КАРТА КУРСА

- **Занятие 1 — «Первый запрос».** Что такое база данных. Таблицы. SELECT * FROM.

- **Занятие 3 — «Сортируем добычу».** ORDER BY, LIMIT. Топ-10 пиратов.
- **Занятие 4 — «Считаем сокровища».** COUNT, SUM, AVG, MAX, MIN.
- **Занятие 5 — «Группируем по кораблям».** GROUP BY, HAVING.
- **Занятие 6 — «Объединяем таблицы».** JOIN, LEFT JOIN. Связи.
- **Занятие 7 — «Добавляем и удаляем».** INSERT, UPDATE, DELETE.
- **Занятие 8 — «Создаём базу».** CREATE TABLE, типы данных, PRIMARY KEY.
- **Занятие 9 — «Большой квест».** Комплексный запрос по всей пиратской базе.
- **Занятие 10 — «Галерея запросов».** Парад проектов. Баджи, жетоны, сертификаты.

📖 ЗАНЯТИЕ 2: ГДЕ ЗОЛОТО?

🎯 Цель

Освоить фильтрацию с WHERE.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Все пираты — хорошо. Но где те, у кого золота больше 100? WHERE — это твой фильтр!»
· 5–15 мин: WHERE. Условия: =, >, <, >=, <=, <>.
· 15–25 мин: AND, OR. Комбинирование условий.
· 25–40 мин: LIKE. Поиск по шаблону: имена на «К», звания с «капитан».
· 40–55 мин: Практика. «Найди пиратов с золотом > 100 И званием 'капитан'».
· 55–60 мин: Итоги. Бадж «Искатель Золота».

🛠️ Практика

```sql
SELECT * FROM pirates WHERE gold > 100;
SELECT name FROM pirates WHERE rank = 'капитан';
SELECT * FROM pirates WHERE gold > 50 AND rum < 10;
SELECT * FROM pirates WHERE rank = 'боцман' OR rank = 'штурман';
SELECT name FROM pirates WHERE name LIKE 'К%';
SELECT * FROM treasures WHERE type = 'золото' AND value > 500;
```

🎮 Игровая механика

Квест «Найди предателя». Наставник описывает подозреваемого: «Золота меньше 20, ром больше 15, звание — юнга». Дети пишут WHERE-запрос.

🏠 Миссия

· 🥉 Юнга: Найти пиратов с золотом > 50.
· 🥈 Боцман: Найти капитанов с золотом > 200.
· 🥇 Штурман: Найти все сокровища типа «золото» ценностью > 300 ИЛИ типа «оружие».

📖 ЗАНЯТИЕ 3: СОРТИРУЕМ ДОБЫЧУ

🎯 Цель

Освоить сортировку и ограничение вывода.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Кто самый богатый пират? А кто самый бедный? ORDER BY расставит всех по порядку!»
· 5–15 мин: ORDER BY. По возрастанию (ASC) и убыванию (DESC).
· 15–25 мин: LIMIT. Топ-5, топ-10.
· 25–40 мин: Сортировка по нескольким полям.
· 40–55 мин: Практика. «Топ-10 самых богатых пиратов». «3 самых бедных юнги».
· 55–60 мин: Итоги. Бадж «Сортировщик».

🛠️ Практика

```sql
SELECT name, gold FROM pirates ORDER BY gold DESC;
SELECT name, rum FROM pirates ORDER BY rum ASC LIMIT 5;
SELECT name, gold, rum FROM pirates ORDER BY gold DESC, rum ASC;
SELECT name, gold FROM pirates WHERE rank = 'юнга' ORDER BY gold ASC LIMIT 3;
SELECT * FROM treasures ORDER BY value DESC LIMIT 10;
```

🎮 Игровая механика

Соревнование «Кто быстрее найдёт топ?». Наставник: «Топ-5 пиратов по рому». Дети пишут запрос. Кто быстрее — жетон.

🏠 Миссия

· 🥉 Юнга: Отсортировать пиратов по золоту (от большего к меньшему).
· 🥈 Боцман: Вывести топ-5 пиратов по золоту.
· 🥇 Штурман: Вывести топ-3 капитанов по золоту и топ-3 юнг по рому.

📖 ЗАНЯТИЕ 4: СЧИТАЕМ СОКРОВИЩА

🎯 Цель

Освоить агрегатные функции.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Сколько всего золота у моего экипажа? А средний ром на пирата? Считаем!»
· 5–15 мин: COUNT — количество записей.
· 15–25 мин: SUM — сумма. AVG — среднее.
· 25–40 мин: MAX и MIN — максимум и минимум.
· 40–55 мин: Практика. Считаем статистику по пиратской базе.
· 55–60 мин: Итоги. Бадж «Счетовод».

🛠️ Практика

```sql
SELECT COUNT(*) FROM pirates;
SELECT SUM(gold) FROM pirates;
SELECT AVG(gold) FROM pirates;
SELECT MAX(gold) FROM pirates;
SELECT MIN(gold) FROM pirates;
SELECT COUNT(*) FROM pirates WHERE rank = 'капитан';
SELECT SUM(value) FROM treasures WHERE type = 'золото';
SELECT AVG(rum) FROM pirates WHERE ship_id = 1;
```

🎮 Игровая механика

Викторина «Угадай число». Наставник: «Сколько всего пиратов в базе?» Дети пишут COUNT. Кто первый с правильным ответом — жетон.

🏠 Миссия

· 🥉 Юнга: Посчитать всех пиратов.
· 🥈 Боцман: Найти сумму золота и средний ром.
· 🥇 Штурман: Для каждого корабля посчитать сумму золота его экипажа.

📖 ЗАНЯТИЕ 5: ГРУППИРУЕМ ПО КОРАБЛЯМ

🎯 Цель

Освоить GROUP BY и HAVING.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Сколько пиратов на каждом корабле? А у какого корабля общее золото больше 500? Группируем!»
· 5–15 мин: GROUP BY. Группировка по столбцу.
· 15–25 мин: Агрегатные функции с GROUP BY.
· 25–40 мин: HAVING. Фильтрация групп.
· 40–55 мин: Практика. Статистика по кораблям.
· 55–60 мин: Итоги. Бадж «Группировщик».

🛠️ Практика

```sql
SELECT ship_id, COUNT(*) FROM pirates GROUP BY ship_id;
SELECT ship_id, SUM(gold) FROM pirates GROUP BY ship_id;
SELECT ship_id, AVG(rum) FROM pirates GROUP BY ship_id;
SELECT ship_id, SUM(gold) FROM pirates GROUP BY ship_id HAVING SUM(gold) > 500;
SELECT type, COUNT(*), SUM(value) FROM treasures GROUP BY type;
```

🎮 Игровая механика

Соревнование «Аналитик». Наставник: «Какой корабль принёс больше всего золота?» Дети пишут запрос с GROUP BY и ORDER BY. Кто первый — жетон.

🏠 Миссия

· 🥉 Юнга: Посчитать пиратов на каждом корабле.
· 🥈 Боцман: Найти сумму золота по кораблям.
· 🥇 Штурман: Найти корабли, где среднее золото экипажа > 100.

📖 ЗАНЯТИЕ 6: ОБЪЕДИНЯЕМ ТАБЛИЦЫ

🎯 Цель

Освоить JOIN для связи таблиц.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Пираты в одной таблице. Корабли в другой. Как узнать, кто на каком корабле? JOIN!»
· 5–15 мин: Концепция связей. PRIMARY KEY и FOREIGN KEY.
· 15–30 мин: INNER JOIN. Только совпадающие записи.
· 30–45 мин: LEFT JOIN. Все записи из левой таблицы.
· 45–55 мин: Практика. Объединяем пиратов с кораблями и сокровищами.
· 55–60 мин: Итоги. Бадж «Объединитель».

🛠️ Практика

```sql
SELECT pirates.name, ships.name 
FROM pirates 
JOIN ships ON pirates.ship_id = ships.id;

SELECT pirates.name, treasures.name, treasures.value 
FROM pirates 
JOIN treasures ON pirates.id = treasures.pirate_id;

SELECT ships.name, COUNT(pirates.id), SUM(pirates.gold) 
FROM ships 
LEFT JOIN pirates ON ships.id = pirates.ship_id 
GROUP BY ships.name;
```

🎮 Игровая механика

Квест «Найди владельца». Наставник: «Найди, кому принадлежит сокровище "Золотой Дублон"». Дети пишут JOIN. Кто первый — жетон.

🏠 Миссия

· 🥉 Юнга: Вывести имена пиратов и названия их кораблей.
· 🥈 Боцман: Вывести имена пиратов и их сокровища.
· 🥇 Штурман: Вывести корабли и сумму золота их экипажа, включая корабли без пиратов.

📖 ЗАНЯТИЕ 7: ДОБАВЛЯЕМ И УДАЛЯЕМ

🎯 Цель

Освоить INSERT, UPDATE, DELETE.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Новый пират вступает в экипаж! Старый уходит на пенсию. Учимся добавлять, менять и удалять!»
· 5–15 мин: INSERT INTO. Добавление записей.
· 15–25 мин: UPDATE. Изменение записей.
· 25–35 мин: DELETE. Удаление записей.
· 35–50 мин: WHERE в UPDATE и DELETE. Защита от удаления всего.
· 50–55 мин: Итоги. Бадж «Капитан Данных».

🛠️ Практика

```sql
INSERT INTO pirates (name, rank, ship_id, gold, rum) 
VALUES ('Юнга Вася', 'юнга', 1, 0, 0);

UPDATE pirates SET gold = gold + 50 WHERE name = 'Юнга Вася';
UPDATE pirates SET rum = rum + 5 WHERE ship_id = 1;

DELETE FROM pirates WHERE name = 'Юнга Вася';
DELETE FROM treasures WHERE value < 10;
```

🎮 Игровая механика

Игра «Пиратская биржа». Наставник: «Капитан Термикс повышает всем юнгам золото на 10!» Дети пишут UPDATE. «Предатель удалён из базы!» — DELETE.

🏠 Миссия

· 🥉 Юнга: Добавить нового пирата.
· 🥈 Боцман: Добавить пирата и изменить его золото.
· 🥇 Штурман: Повысить ранг всем пиратам с золотом > 200 до «ветеран».

📖 ЗАНЯТИЕ 8: СОЗДАЁМ БАЗУ

🎯 Цель

Освоить CREATE TABLE и проектирование баз данных.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Готовая база — хорошо. Но своя — лучше! Сегодня создаём базу с нуля!»
· 5–15 мин: CREATE TABLE. Типы данных: INTEGER, TEXT, REAL.
· 15–25 мин: PRIMARY KEY. Уникальный идентификатор.
· 25–40 мин: FOREIGN KEY. Связи между таблицами.
· 40–55 мин: Создаём свою базу «Пиратский Архив».
· 55–60 мин: Итоги. Бадж «Архитектор Баз».

🛠️ Практика

```sql
CREATE TABLE islands (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    region TEXT,
    danger_level INTEGER
);

CREATE TABLE raids (
    id INTEGER PRIMARY KEY,
    ship_id INTEGER,
    island_id INTEGER,
    date TEXT,
    gold_stolen INTEGER,
    FOREIGN KEY (ship_id) REFERENCES ships(id),
    FOREIGN KEY (island_id) REFERENCES islands(id)
);

INSERT INTO islands (name, region, danger_level) 
VALUES ('Остров Черепа', 'Южные моря', 5);
```

🎮 Игровая механика

Конкурс «Спроектируй базу». Наставник: «Придумай таблицу для магических артефактов. Какие столбцы?» Дети предлагают структуру. Лучшая — жетон.
