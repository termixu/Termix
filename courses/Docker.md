---

## 🧠 ФИЛОСОФИЯ КУРСА

Обычный подход: Docker → контейнеры → образы → сложно → «работает на моей машине».

Подход Termix Academy: Docker → грузовой отсек → упаковал приложение → запустил где угодно → «работает везде».

Docker — это технология контейнеризации. Она позволяет упаковать приложение со всеми зависимостями в изолированный контейнер и запустить его на любом сервере. Termix учит детей упаковывать свои проекты (Python-бота, HTML-сайт, Node.js-сервер) в Docker-контейнеры и запускать их одной командой.

---

## 🎯 ЧЕМУ НАУЧИТСЯ РЕБЁНОК

- Понимать, что такое контейнеризация и зачем она нужна.
- Устанавливать Docker.
- Запускать готовые контейнеры (nginx, python, node).
- Писать Dockerfile для своих проектов.
- Собирать образы (docker build).
- Запускать контейнеры с флагами (порты, тома, переменные).
- Работать с Docker Compose (много контейнеров).
- Понимать, как Docker используется в реальной разработке.

## 📖 ЗАНЯТИЕ 1: ПЕРВЫЙ КОНТЕЙНЕР

### 🎯 Цель

Понять, что такое Docker, установить и запустить первый контейнер.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «Бип! Добро пожаловать в грузовой отсек! Docker — это капсула, в которую мы упаковываем приложение. Погнали!»
- 5–15 мин: Что такое контейнеризация. Проблема «работает на моей машине».
- 15–30 мин: Установка Docker. Проверка: `docker --version`.
- 30–45 мин: `docker run hello-world`. Что произошло? Скачался образ, запустился контейнер.
- 45–55 мин: `docker ps`, `docker ps -a`. Список контейнеров.
- 55–60 мин: Итоги. Бадж «Первый Контейнер».

### 🛠️ Практика

```bash
docker --version                        # проверить установку
docker run hello-world                  # первый контейнер
docker ps                               # запущенные контейнеры
docker ps -a                            # все контейнеры (включая остановленные)
docker rm <container_id>                # удалить контейнер
docker rmi hello-world                  # удалить образ
```

📖 ЗАНЯТИЕ 2: ГОТОВЫЕ ОБРАЗЫ

🎯 Цель

Освоить Docker Hub и флаги запуска контейнеров.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Docker Hub — это рынок готовых капсул! Бери что хочешь: веб-сервер, базу данных, целый Linux!»
· 5–15 мин: Docker Hub. Поиск образов.
· 15–25 мин: nginx. Запуск веб-сервера в контейнере.
· 25–40 мин: Флаги: -d (фон), -p (порты), --name (имя).
· 40–55 мин: docker logs, docker exec. Заглянуть внутрь контейнера.
· 55–60 мин: Итоги. Бадж «Капитан Образов».

🛠️ Практика

```bash
docker pull nginx                         # скачать образ
docker run -d -p 8080:80 --name my-nginx nginx   # запустить
# Открыть в браузере: http://localhost:8080
docker ps                                 # проверить
docker logs my-nginx                      # посмотреть логи
docker exec -it my-nginx bash             # зайти внутрь контейнера
# внутри: ls, cat /usr/share/nginx/html/index.html
# exit — выйти
docker stop my-nginx                      # остановить
docker rm my-nginx                        # удалить
```

🎮 Игровая механика

Соревнование «Запусти зоопарк». Наставник: «Запусти nginx на порту 1234, проверь что работает, зайди внутрь, найди файл index.html». Кто первый — жетон.

🏠 Миссия

· 🥈 Боцман: Запустить nginx с именем и на своём порту, зайти внутрь.
· 🥇 Штурман: Запустить python-контейнер и выполнить внутри python-скрипт.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Готовые образы — хорошо. Но свои — лучше! Dockerfile — это чертёж твоей капсулы!»
· 5–15 мин: Что такое Dockerfile. Инструкции.
· 15–25 мин: FROM, COPY, RUN, CMD. Базовая структура.
· 25–45 мин: Создаём образ с HTML-сайтом.
· 45–55 мин: docker build, docker run. Сборка и запуск.
· 55–60 мин: Итоги. Бадж «Инженер Образов».

🛠️ Практика

Ребёнок создаёт папку my-site, внутри index.html и Dockerfile:

index.html:

```html
<h1>Бип! Termix Academy!</h1>
<p>Этот сайт запущен в Docker!</p>
```

Dockerfile:

```dockerfile
FROM nginx:alpine
COPY index.html /usr/share/nginx/html/
```

```bash
docker build -t my-site .                # собрать образ
docker run -d -p 8888:80 my-site         # запустить
# Открыть http://localhost:8888
```

📖 ЗАНЯТИЕ 4: СЛОИ И КЭШ

🎯 Цель

Понять, как работают слои Docker, и оптимизировать Dockerfile.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Каждый слой — как палуба корабля. Если одна палуба не меняется — мы не перестраиваем весь корабль!»
· 5–15 мин: Слои Docker. Как работает кэш.
· 15–25 мин: docker history. Посмотреть слои образа.
· 25–40 мин: Оптимизация: порядок инструкций, .dockerignore.
· 40–55 мин: Пересобираем образ с оптимизацией.
· 55–60 мин: Итоги. Бадж «Оптимизатор».

🛠️ Практика

```bash
docker history my-site                    # посмотреть слои
docker build --no-cache -t my-site .     # пересобрать без кэша
```

Оптимизированный Dockerfile:

```dockerfile
FROM nginx:alpine
COPY index.html /usr/share/nginx/html/
# COPY идёт после FROM, чтобы изменения в index.html не сбрасывали кэш nginx
```

Создание .dockerignore:

```
node_modules
*.log
.git
```

🎮 Игровая механика

Игра «Сколько слоёв?». Наставник показывает Dockerfile. Дети угадывают, сколько слоёв будет у образа. Кто ближе — жетон.

🏠 Миссия

· 🥉 Юнга: Посмотреть слои своего образа через docker history.
· 🥈 Боцман: Пересобрать образ с .dockerignore.
· 🥇 Штурман: Оптимизировать Dockerfile так, чтобы кэшировалось больше слоёв.

📖 ЗАНЯТИЕ 5: ТОМА — ТРЮМЫ

🎯 Цель

Освоить Docker Volumes для сохранения данных.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Контейнер умер — данные пропали. Так не пойдёт! Тома — это трюмы, где сокровища лежат вечно!»
· 5–15 мин: Проблема: данные в контейнере не сохраняются.
· 15–25 мин: -v. Примонтировать папку с хоста.
· 25–40 мин: Named volumes. docker volume create.
· 40–55 мин: Практика. Nginx с томом для сайта.
· 55–60 мин: Итоги. Бадж «Хранитель Трюмов».

🛠️ Практика

```bash
mkdir ~/my-website
echo "<h1>Бип!</h1>" > ~/my-website/index.html
docker run -d -p 9090:80 -v ~/my-website:/usr/share/nginx/html nginx
# Изменить index.html на хосте → обновить браузер → изменения видны

docker volume create termix-data
docker run -d -v termix-data:/data alpine sleep infinity
```

🎮 Игровая механика

Игра «Сохрани сокровище». Наставник: «Создай том, запиши туда файл, удали контейнер, создай новый с тем же томом, проверь что файл на месте». Кто первый — жетон.

🏠 Миссия

· 🥉 Юнга: Запустить nginx с томом.
· 🥈 Боцман: Создать named volume и использовать его.
· 🥇 Штурман: Настроить том для базы данных и проверить сохранность данных.

📖 ЗАНЯТИЕ 6: COMPOSE — ЭСКАДРА

🎯 Цель

Освоить Docker Compose для запуска нескольких контейнеров.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Один контейнер — лодка. Много контейнеров — эскадра! Compose управляет целым флотом!»
· 5–15 мин: Что такое Docker Compose. docker-compose.yml.
· 15–30 мин: Структура файла: services, ports, volumes.
· 30–50 мин: Запускаем связку: сайт (nginx) + API (python).
· 50–55 мин: docker-compose up, docker-compose down.
· 55–60 мин: Итоги. Бадж «Адмирал».

🛠️ Практика

Ребёнок создаёт docker-compose.yml:

```yaml
version: '3'
services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - ./site:/usr/share/nginx/html
  
  api:
    image: python:alpine
    command: python -m http.server 8000
    ports:
      - "8000:8000"
```

```bash
docker-compose up -d                    # запустить эскадру
docker-compose ps                        # посмотреть статус
docker-compose logs                      # логи всех контейнеров
docker-compose down                      # остановить и удалить
```

🎮 Игровая механика

Соревнование «Подними флот». Наставник: «Создай compose-файл с nginx и вторым контейнером. Запусти. Покажи что оба работают». Кто первый — жетон.

🏠 Миссия

· 🥉 Юнга: Запустить два контейнера через Compose.
· 🥈 Боцман: Добавить том в Compose.
· 🥇 Штурман: Сделать Compose из трёх сервисов с зависимостями (depends_on).

📖 ЗАНЯТИЕ 7: МОЙ ПРОЕКТ В КОНТЕЙНЕРЕ

🎯 Цель

Упаковать свой реальный проект в Docker.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Сегодня ты — главный инженер! Упакуй свой проект в контейнер и отправь в плавание!»
· 5–10 мин: Выбор проекта. Telegram-бот, HTML-сайт, Python-скрипт.
· 10–60 мин: Написание Dockerfile, сборка, тестирование.
· 60–65 мин: Сохранение и экспорт.

🛠️ Практика

Пример для Telegram-бота на Python:

Dockerfile:

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "bot.py"]
```

```bash
docker build -t termix-bot .
docker run -d --name termix-bot termix-bot
docker logs termix-bot
```

«Мозговой штурм с Termix'ом». Termix задаёт вопросы: «Что упаковываешь? Какие зависимости? Как проверить что работает?»
