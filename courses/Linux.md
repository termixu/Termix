## 🧠 ФИЛОСОФИЯ КУРСА

Обычный подход: Linux → чёрный экран → сложно → «я сломаю систему».

Подход Termix Academy: Linux → подводная лодка → ты — капитан → «пеленгую, погружаюсь, всплываю».

Linux — это операционная система, на которой работает весь интернет. Серверы, облака, суперкомпьютеры, Raspberry Pi — всё на Linux. Termix учит детей управлять системой через терминал: пользователи, права, процессы, пакеты, сеть. К концу курса ребёнок может поднять свой сайт на настоящем сервере.

## 📖 ЗАНЯТИЕ 1: ПОГРУЖЕНИЕ

### 🎯 Цель

Понять, что такое Linux, освоить базовые команды и структуру файловой системы.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix: «Бип! Добро пожаловать на борт подводной лодки! Linux — это твоя система жизнеобеспечения. Погнали!»
- 5–15 мин: Что такое Linux. Ubuntu, Debian, серверы. Виртуальная машина или WSL.
- 15–25 мин: Файловая система. `/` — корень. `/home` — каюты. `/etc` — приборы. `/var` — склады.
- 25–45 мин: Базовые команды: `pwd`, `ls`, `cd`, `mkdir`, `touch`, `rm`, `cp`, `mv`.
- 45–55 мин: `man` — документация. `--help` — справка.
- 55–60 мин: Итоги. Бадж «Погружение».

### 🛠️ Практика

```bash
pwd                          # где я?
ls -la /                     # что в корне системы?
cd /home                     # перейти в домашнюю папку
mkdir submarine              # создать папку
cd submarine
touch logbook.txt            # создать файл
echo "Погружение начато" > logbook.txt
cat logbook.txt
man ls                       # документация по ls
```

Игра «Обойди отсеки». Наставник: «Перейди в /etc, посмотри что там, вернись в /home». Дети выполняют. Кто первый — жетон.

📖 ЗАНЯТИЕ 2: КАПИТАНСКАЯ РУБКА

🎯 Цель

Освоить управление пользователями, группами и правами доступа.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «На лодке у каждого свой пост. Капитан, штурман, механик. Сегодня создаём экипаж!»
· 5–15 мин: whoami, id, users. Кто я и кто ещё на борту.
· 15–25 мин: useradd, passwd, usermod. Создание пользователей.
· 25–40 мин: chmod, chown. Права доступа: rwx, цифры 755.
· 40–55 мин: sudo. Капитанские полномочия.
· 55–60 мин: Итоги. Бадж «Капитанская Рубка».

🛠️ Практика

```bash
whoami                       # кто я?
id                           # мои группы
sudo useradd -m pirate1      # создать пользователя (нужен sudo)
sudo passwd pirate1          # задать пароль
su pirate1                   # переключиться на пользователя
exit                         # вернуться
ls -la                       # посмотреть права
chmod 600 secret.txt         # только владелец может читать и писать
chmod 755 script.sh          # владелец — всё, остальные — читать и запускать
sudo chown pirate1 file.txt  # сменить владельца
```

🎮 Игровая механика

Игра «Собери экипаж». Наставник: «Создай трёх пользователей: captain, navigator, mechanic. Дай им пароли. Проверь, что они могут зайти». Дети выполняют. Кто первый — жетон.

📖 ЗАНЯТИЕ 4: ПЕРИСКОП

🎯 Цель

Освоить управление процессами.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Перископ поднят! Вижу процессы. Одни полезные, другие — вражеские. Учимся управлять!»
· 5–15 мин: ps, ps aux. Список процессов.
· 15–25 мин: top, htop. Интерактивный мониторинг.
· 25–40 мин: kill, killall, сигналы. Завершение процессов.
· 40–55 мин: &, jobs, fg, bg. Фоновые процессы.
· 55–60 мин: Итоги. Бадж «Перископ».

🛠️ Практика

```bash
ps aux | grep firefox           # найти процесс firefox
top                              # мониторинг (q — выход)
htop                             # красивый мониторинг
sleep 1000 &                     # запустить в фоне
jobs                             # посмотреть фоновые задачи
fg %1                            # вернуть на передний план
kill %1                          # завершить задачу
kill -9 PID                      # жёсткое завершение процесса
```

🎮 Игровая механика

Игра «Охота на процесс». Наставник запускает sleep 9999 &. Дети должны найти этот процесс через ps, узнать его PID и убить через kill. Кто первый — жетон.

```bash

# Ctrl+O — сохранить, Ctrl+X — выйти

# i — режим вставки
# Написать текст
# Esc — выйти из режима вставки
# :w — сохранить
# :q — выйти

nano ~/.bashrc               # редактировать настройки терминала
# Добавить: alias ll='ls -la'

```

📖 ЗАНЯТИЕ 7: ГЕРМЕТИЗАЦИЯ

🎯 Цель

Освоить файрвол и базовую безопасность.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Пробоина! Враг пытается проникнуть! Файрвол — это герметичные переборки. Закрываем люки!»
· 5–15 мин: Что такое файрвол. Порты. Открытые и закрытые.
· 15–30 мин: ufw. Uncomplicated Firewall. Включение, правила.
· 30–45 мин: ss, netstat. Прослушиваемые порты.
· 45–55 мин: SSH-ключи. Безопасный вход без пароля.
· 55–60 мин: Итоги. Бадж «Герметизация».

🛠️ Практика

```bash
sudo ufw status                  # статус файрвола
sudo ufw enable                  # включить
sudo ufw allow 22                # разрешить SSH
sudo ufw allow 80                # разрешить HTTP
sudo ufw allow 443               # разрешить HTTPS
sudo ufw deny 23                 # запретить Telnet
sudo ufw status verbose          # подробный статус
ss -tlnp                         # послушать порты
ssh-keygen                       # создать SSH-ключ
ssh-copy-id user@server          # скопировать ключ на сервер
```

🎮 Игровая механика

Квест «Защити лодку». Наставник: «Открой порт 80, закрой порт 23, проверь статус». Дети выполняют цепочку команд.

🏠 Миссия

· 🥉 Юнга: Включить ufw и разрешить SSH.
· 🥈 Боцман: Настроить правила для 3 портов.
· 🥇 Штурман: Создать SSH-ключ и объяснить, зачем он нужен.
