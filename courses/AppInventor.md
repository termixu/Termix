> **Версия:** 1.0
>
> **Возраст:** 10–14 лет
>
> **Длительность:** 8 занятий по 60–75 минут
>
> **Проводник:** Termix — кибер-пиратский кот, Хранитель Ядра
>
> **Принцип:** Не учить синтаксис, а собирать приложения из блоков. Не писать код, а создавать настоящие Android-приложения для своих проектов.

---

## 🧠 ФИЛОСОФИЯ КУРСА

Обычный подход: мобильная разработка → Java/Kotlin → сложно → «я не смогу сделать приложение».

Подход Termix Academy: App Inventor → визуальные блоки → собрал как пазл → запустил на телефоне → «моё приложение работает!».

MIT App Inventor — это среда для создания Android-приложений без написания кода. Всё в браузере. Компоненты перетаскиваются на экран телефона. Логика собирается из визуальных блоков, как в Scratch. Termix учит детей создавать настоящие приложения: от простой «Нажималки» до мини-игр с кнопками и счётом.

---

## 🎯 ЧЕМУ НАУЧИТСЯ РЕБЁНОК

- Понимать структуру мобильного приложения.
- Работать в визуальной среде App Inventor.
- Добавлять компоненты: кнопки, текст, изображения, датчики.
- Программировать логику через визуальные блоки.
- Использовать переменные и условия.
- Работать с экранами (многостраничные приложения).
- Тестировать приложение на реальном телефоне.
- Собирать APK-файл и делиться с друзьями.

## 📖 ЗАНЯТИЕ 1: ПЕРВАЯ КНОПКА

### 🎯 Цель

Освоить интерфейс App Inventor, создать первое приложение с кнопкой.

### ⏱️ Тайминг (60–75 минут)

- 0–5 мин: Termix: «Бип! Добро пожаловать на борт карманного корабля! App Inventor — это верфь, где рождаются мобильные приложения. Погнали!»
- 5–15 мин: Вход через MIT App Inventor (ai2.appinventor.mit.edu). Интерфейс: Designer и Blocks.
- 15–25 мин: Режим Designer. Добавление компонентов: Button, Label.
- 25–40 мин: Режим Blocks. Собираем блок: When Button.Click → set Label.Text to «Бип!».
- 40–50 мин: Тестирование на телефоне (приложение MIT AI2 Companion) или эмуляторе.
- 50–55 мин: Сохранение проекта. Итоги. Бадж «Первая кнопка».

### 🛠️ Практика

Ребёнок открывает App Inventor. Создаёт новый проект «PirateApp». В Designer перетаскивает Button (текст «Нажми меня!») и Label (пустой). Переходит в Blocks. Собирает: when Button1.Click → set Label1.Text to «Бип! Привет, юнга!». Запускает на телефоне через Companion. Нажимает кнопку — видит текст.

### 🎮 Игровая механика

Игра «Кто быстрее?». Наставник: «Добавь вторую кнопку, которая меняет текст на другой». Кто первый — жетон.

### 🏠 Миссия

- 🥉 Юнга: Кнопка меняет текст Label.
- 🥈 Боцман: Две кнопки с разными текстами.
- 🥇 Штурман: Кнопка меняет цвет фона экрана.

## 📖 ЗАНЯТИЕ 2: ПИРАТСКИЙ КАЛЬКУЛЯТОР

### 🎯 Цель

Освоить переменные и ввод данных.

### ⏱️ Тайминг (60–75 минут)

- 0–5 мин: Termix: «Считаем золото! Переменные — это сундучки для чисел. Вводим данные, считаем, выводим результат!»
- 5–15 мин: Компонент TextBox (ввод текста). Переменные в Blocks.
- 15–30 мин: Сложение. Вводим два числа, нажимаем кнопку — результат.
- 30–45 мин: Все 4 действия: +, -, *, /. Выбор действия через выпадающий список.
- 45–55 мин: Форматирование вывода. «Ответ: 42 золотых дублонов».
- 55–60 мин: Итоги. Бадж «Счетовод».

### 🛠️ Практика

Ребёнок добавляет два TextBox (Число 1, Число 2), Button «Сложить», Label (результат). В Blocks:

- Initialize global variable (result) to 0.
- When Button.Click → set global result to TextBox1.Text + TextBox2.Text.
- Set Label.Text to join «Результат: » get global result.

Для 4 действий добавляет ListPicker с вариантами «+», «-», «*», «/» и условие if-else.

### 🎮 Игровая механика

Соревнование «Калькулятор». Наставник даёт 5 примеров. Дети считают в своём приложении. Кто быстрее — жетон.

### 🏠 Миссия

- 🥉 Юнга: Сложение двух чисел.
- 🥈 Боцман: Все 4 действия.
- 🥇 Штурман: Добавить проверку: если деление на ноль — вывести ошибку.

## 📖 ЗАНЯТИЕ 3: ГАЛЕРЕЯ TERMIX'А

### 🎯 Цель

Освоить работу с изображениями.

### ⏱️ Тайминг (60–75 минут)

- 0–5 мин: Termix: «Галерея! Показываем мои фотографии. Листаем вперёд-назад!»
- 5–15 мин: Компонент Image. Загрузка картинок.
- 15–30 мин: Кнопки «Вперёд» и «Назад». Переключение картинок.
- 30–45 мин: Список картинок. Переменная-счётчик.
- 45–55 мин: Подпись к картинке. Название под фото.
- 55–60 мин: Итоги. Бадж «Галерист».

### 🛠️ Практика

Ребёнок загружает 4 картинки Termix'а. Добавляет Image, две Button («←», «→»), Label. Переменная currentIndex = 1. Список картинок. When Button «→» → increment currentIndex, set Image to list[currentIndex]. Проверка границ: если currentIndex > длина списка → currentIndex = 1.

### 🎮 Игровая механика

Конкурс «Чья галерея красивее?». Дети загружают свои картинки. Голосование стикерами.

### 🏠 Миссия

- 🥉 Юнга: Показывать одну картинку.
- 🥈 Боцман: Переключение вперёд-назад.
- 🥇 Штурман: Автоматическое слайд-шоу (Timer + смена каждые 2 секунды).

## 📖 ЗАНЯТИЕ 4: МИНИ-ИГРА — КЛИКЕР

### 🎯 Цель

Освоить счётчики и таймеры.

### ⏱️ Тайминг (60–75 минут)

- 0–5 мин: Termix: «Кликер! Нажимай на меня как можно быстрее. Таймер тикает. Счёт растёт. Играем!»
- 5–15 мин: Счётчик кликов. Переменная score.
- 15–30 мин: Таймер. Countdown 10 секунд.
- 30–45 мин: Игра «Кликер». За 10 секунд накликать максимум.
- 45–55 мин: Рекорд. Сохранение лучшего результата.
- 55–60 мин: Итоги. Бадж «Кликер».

### 🛠️ Практика

Ребёнок добавляет Button (большая, по центру), Label (счёт), Label (таймер). Переменные: score, timeLeft = 10. При клике на кнопку: score += 1, обновить Label. Timer: каждую секунду timeLeft -= 1, обновить Label. Когда timeLeft = 0 → завершить игру, показать результат.

### 🎮 Игровая механика

Турнир «Кликер». Дети играют в кликеры друг друга. Топ-3 — жетоны.

### 🏠 Миссия

- 🥉 Юнга: Счётчик кликов.
- 🥈 Боцман: Добавить таймер на 10 секунд.
- 🥇 Штурман: Сохранять рекорд между запусками (TinyDB).

## 📖 ЗАНЯТИЕ 5: КОМПАС — НАВИГАТОР

### 🎯 Цель

Освоить датчики устройства.

### ⏱️ Тайминг (60–75 минут)

- 0–5 мин: Termix: «Компас! Телефон знает, куда ты смотришь. Датчики — это магия!»
- 5–15 мин: Компонент OrientationSensor. AccelerometerSensor.
- 15–30 мин: Компас. Поворачиваешь телефон — стрелка двигается.
- 30–45 мин: Шагомер. Считаем шаги.
- 45–55 мин: Уровень. Пузырьковый уровень (как в строительстве).
- 55–60 мин: Итоги. Бадж «Навигатор».

### 🛠️ Практика

Ребёнок добавляет OrientationSensor. Создаёт Image (стрелка). При изменении OrientationSensor.Angle → поворачивает стрелку на этот угол. Добавляет AccelerometerSensor. При тряске (Shaking) → Termix говорит «Бип! Не тряси меня!».

### 🎮 Игровая механика

Игра «Найди север». Наставник: «Поверни телефон так, чтобы стрелка показала строго вверх. У кого точнее — жетон».

### 🏠 Миссия

- 🥈 Боцман: Добавить реакцию на тряску.
- 🥇 Штурман: Сделать шагомер.

## 📖 ЗАНЯТИЕ 6: КАРТА СОКРОВИЩ

### 🎯 Цель

Освоить работу с несколькими экранами.

### ⏱️ Тайминг (60–75 минут)

- 0–5 мин: Termix: «Несколько экранов! Как страницы в книге. Главное меню, настройки, игра!»

- 15–30 мин: Главное меню → Экран игры → Экран результатов.

- Screen3 (Результат): счёт, кнопка «Заново».

Переход: when Button.Click → open another screen Screen2.

### 🎮 Игровая механика
