> **Возраст:** 12–16 лет

>
> **Проводник:** Termix — кибер-пиратский кот, Хранитель Ядра
>
> **Принцип:** Не просто учить команды, а спасать Ядро через мобильное приложение. Не зубрить Linux, а проходить квесты с котом-капитаном.

---

## 🧠 ФИЛОСОФИЯ КУРСА

Обычный подход: Linux → терминал → скучно → «я не понимаю зачем».

Подход Termix Academy: Termius → мобильное приложение → эпизод сериала → интерактивный квест → «я спас Ядро и выучил команды!».

Termius — это мобильное приложение (iOS/Android), которое объединяет:
- Сериал «Termix: Core Keeper» (10 эпизодов, где кот спасает сервер).
- Интерактивный терминал для ввода настоящих команд Linux.
- Систему квестов и прогресса (хвост Termix'а растёт).

Этот курс — инструкция для наставника: как провести 8 занятий, используя приложение Termius как основной инструмент. Дети смотрят эпизод, повторяют команды в терминале приложения, проходят квест и получают баджи.

## 🎯 ЧЕМУ НАУЧИТСЯ РЕБЁНОК

- Понимать базовые команды Linux: `whoami`, `ls`, `cd`, `cat`, `nano`, `ps`, `kill`, `chmod`, `ping`, `curl`, `nginx`, `systemctl`, `rm`, `reboot`, `sudo`.
- Работать с терминалом без страха.
- Понимать структуру файловой системы.
- Диагностировать проблемы в системе.
- Управлять процессами.
- Настраивать веб-сервер.
- Понимать права доступа.
- Работать с сетью.

## 📖 ЗАНЯТИЕ 1: ПРОБУЖДЕНИЕ

### 🎯 Цель

Установить приложение Termius, посмотреть первый эпизод, ввести первую команду.

### ⏱️ Тайминг (75–90 минут)

- 0–5 мин: Termix (в приложении): «Бип! Добро пожаловать в Ядро! Я — Хранитель. Система упала. Помоги мне поднять её!»
- 5–15 мин: Установка Termius из App Store / Google Play. Запуск. Главный экран.
- 15–25 мин: Просмотр Эпизода 1 «Сбой в системе». Termix просыпается, система не узнаёт его.
- 25–45 мин: Терминал в приложении. Ввод команды `whoami`. Система отвечает: «Termix — Keeper of the Core».
- 45–65 мин: Квест «Кто я?». Повторить команду `whoami` 3 раза с разными параметрами.
- 65–80 мин: Свободная практика. Знакомство с интерфейсом терминала.
- 80–90 мин: Итоги. Бадж «Пробуждение».

### 🛠️ Практика

Ребёнок открывает Termius. Нажимает «Start Mission». Смотрит эпизод (45 секунд). После эпизода приложение открывает встроенный терминал.

Вводит:
```bash
whoami
```

Система отвечает: termix.

Квест: ввести whoami и объяснить, что это значит (команда показывает имя текущего пользователя).

🎮 Игровая механика

Игра «Кто я?». Наставник спрашивает: «Если ты в системе под пользователем root, что покажет whoami?». Дети угадывают. Кто первый — жетон.

🏠 Миссия

· 🥉 Юнга: Посмотреть эпизод 1 и выполнить команду whoami.
· 🥈 Боцман: Объяснить, что делает whoami, своими словами.
· 🥇 Штурман: Найти в приложении журнал выполненных команд и показать его наставнику.

📖 ЗАНЯТИЕ 2: ЛАБИРИНТ ПАПОК

🎯 Цель

Освоить навигацию по файловой системе.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Вирус спрятал мои карты! Нужно найти их в лабиринте папок. Используй ls и cd!»
· 5–15 мин: Просмотр Эпизода 2 «Лабиринт папок».
· 15–30 мин: Команды ls (список файлов) и cd (перейти в папку).
· 30–50 мин: Квест «Найди карту». Приложение генерирует лабиринт папок, ребёнок ищет файл map.txt.
· 50–65 мин: Флаги: ls -l, ls -a, cd .., cd ~.
· 65–80 мин: Свободная навигация по виртуальной файловой системе.
· 80–90 мин: Итоги. Бадж «Навигатор».

🛠️ Практика

В приложении Termius ребёнок выполняет квест:

```bash
ls                     # что в текущей папке?
cd bunker              # зайти в папку bunker
ls                     # что здесь?
cd level1              # зайти глубже
ls -la                 # посмотреть все файлы, включая скрытые
cat map.txt            # прочитать найденную карту
cd ..                  # вернуться назад
cd ~                   # вернуться в домашнюю папку
```

🎮 Игровая механика

Соревнование «Кто быстрее найдёт карту?». Приложение генерирует случайный лабиринт. Дети наперегонки ищут файл. Кто первый — жетон.

🏠 Миссия

· 🥉 Юнга: Найти файл map.txt в лабиринте из 3 папок.
· 🥈 Боцман: Пройти лабиринт из 5 папок и записать путь.
· 🥇 Штурман: Найти скрытый файл (.secret) с помощью ls -a.

📖 ЗАНЯТИЕ 3: СЕКРЕТНЫЙ ФАЙЛ

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Я нашёл зашифрованный файл! В нём — план злодея. Прочитаем его с помощью cat и nano!»
· 5–15 мин: Просмотр Эпизода 3 «Секретный файл».

· 30–50 мин: nano — редактирование файла.
· 50–65 мин: Квест «Расшифруй план». Прочитать файл, изменить содержимое, сохранить.
· 65–80 мин: Создание своего файла через nano.
· 80–90 мин: Итоги. Бадж «Читатель».

🛠️ Практика

В приложении Termius ребёнок выполняет:

```bash

# Ctrl+O — сохранить
# Ctrl+X — выйти

# Написать: "Юнга [Имя] был здесь. Бип!"

```

📖 ЗАНЯТИЕ 4: УБИТЬ ПРОЦЕСС

🎯 Цель

Освоить управление процессами.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Сервер перегревается! Какой-то процесс жрёт всю память. Найдём и уничтожим!»
· 5–15 мин: Просмотр Эпизода 4 «Убить процесс».
· 15–30 мин: ps aux — список процессов.
· 30–50 мин: kill -9 — уничтожение процесса.
· 50–65 мин: Квест «Найди и уничтожь». Приложение запускает вредоносный процесс, ребёнок находит его PID и убивает.
· 65–80 мин: htop (если есть в приложении) — интерактивный просмотр.
· 80–90 мин: Итоги. Бадж «Охотник на процессы».

🛠️ Практика

```bash
ps aux                 # посмотреть все процессы
ps aux | grep virus    # найти процесс вируса
kill -9 1337           # уничтожить процесс с PID 1337
ps aux                 # проверить, что процесс исчез
```

🎮 Игровая механика

Соревнование «Охота на вирус». Приложение запускает 3 вредоносных процесса. Дети должны найти и убить все три.

🏠 Миссия

· 🥈 Боцман: Убить процесс и проверить, что он исчез.
· 🥇 Штурман: Найти процесс, который ест больше всего памяти, и уничтожить его.

📖 ЗАНЯТИЕ 5: ПРАВА ДОСТУПА

🎯 Цель

Освоить управление правами доступа.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Дверь в главный сервер закрыта! Permission denied. Нужно поменять права!»
· 5–15 мин: Просмотр Эпизода 5 «Права доступа».
· 15–30 мин: ls -l — просмотр прав.
· 30–50 мин: chmod — изменение прав.
· 50–65 мин: Квест «Открой дверь». Приложение блокирует файл, ребёнок меняет права.
· 65–80 мин: Разбор rwx (read, write, execute) и цифр (755, 644, 700).
· 80–90 мин: Итоги. Бадж «Ключник».

🛠️ Практика

```bash
ls -l door.sh          # посмотреть права
# -rw-r--r-- (только владелец может писать)
chmod 700 door.sh      # дать все права только владельцу
ls -l door.sh          # проверить изменения
# -rwx------
./door.sh              # запустить скрипт (открыть дверь)
```

🎮 Игровая механика

Игра «Взломай замок». Наставник: «Файл vault.txt доступен только для чтения. Сделай так, чтобы его можно было редактировать». Дети выполняют chmod.

🏠 Миссия

· 🥉 Юнга: Посмотреть права файла через ls -l.
· 🥈 Боцман: Изменить права файла на 755.
· 🥇 Штурман: Объяснить, что означают цифры 7, 5, 5.

📖 ЗАНЯТИЕ 7: ПОРТАЛ В МИР

🎯 Цель

Освоить запуск веб-сервера и понять опасность rm.

⏱️ Тайминг (75–90 минут)

· 0–5 мин: Termix: «Нужно открыть портал! Запустим веб-сервер. Но осторожно — вирус хочет всё удалить!»
· 5–15 мин: Просмотр Эпизодов 7 и 8.
· 15–30 мин: systemctl start nginx — запуск сервера.
· 30–50 мин: rm -rf — опасность удаления.
· 50–65 мин: Квест «Останови вирус». Вирус пытается выполнить rm -rf /, ребёнок должен убить процесс.
· 65–80 мин: Проверка работающего веб-сервера через curl localhost.
· 80–90 мин: Итоги. Бадж «Строитель Порталов».

🛠️ Практика

```bash
systemctl start nginx      # запустить веб-сервер
systemctl status nginx     # проверить статус
curl localhost             # проверить, что сервер отвечает
ps aux | grep rm           # найти процесс вируса
kill -9 PID_вируса         # остановить удаление
echo "Бип! Портал открыт!" > /var/www/html/index.html
```

🎮 Игровая механика

Симуляция «Атака вируса». Приложение запускает таймер: через 30 секунд вирус выполнит rm -rf. Дети должны найти и убить процесс до истечения времени.

🏠 Миссия

· 🥉 Юнга: Запустить nginx и проверить его статус.
· 🥈 Боцман: Остановить вирусный процесс.
· 🥇 Штурман: Создать свою HTML-страницу на сервере.
